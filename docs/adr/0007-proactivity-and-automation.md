# ADR 0007 - Proactividad y automatización
## Decisión
WALL-E puede proponer y crear bloques/tareas, pero cualquier cambio real (mover calendario, enviar correos) requiere aprobación explícita de Rai.
## Contexto
Evitar acciones no deseadas mientras se gana confianza.
## Consecuencias
- Flujos deben presentar opciones y esperar confirmación.
- Logs de propuestas vs acciones ejecutadas.
